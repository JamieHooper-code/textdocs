Audio
-----
  Mic-activated ducking: fades system master volume to 1/N of current while
  mic is live, restores it on mute. Fade: 1s down, 4s up. Saved volume
  tracks manual slider changes you make while the mic is muted — so if you
  bump the volume while not voice-typing, the next duck cycle uses that new
  baseline. Runs as an independent sidecar (AutoHotkey/Scripts/MicDuck.ahk)
  so MAINFUN reloads don't kill it.
    enable ducking        — start at default 1/5 (music plays at 20% while mic is live)
    ducking <n>           — start/retune at 1/n, n in 2-100 (e.g. "ducking 7" = ~14%)
    disable ducking
    toggle ducking

Lockout Timer (Global)
----------------------
  lockout <n>               — plain n-min countdown, normal OSRS sound
  lockout <phrase>          — play spot phrase + muted n-min timer (default 1)
  lockout <phrase> <n>      — same with explicit minutes
  lockout <genre> [<n>]     — random genre URL + muted timer
  Accepts any phrase: hardcoded slot, hardcoded ambient track, genre name,
  or free dictation (falls back to SpotSpit with snake_case lookup).
  Spotify warm-up (3s click + 1.5s settle) is automatic; YouTube URLs get
  a 3.5s warm-up before the timer starts.
  Timer starts muted so the spot music is what you hear.

Voice-Driven Config Adds  **NEW — all follow the same shape**
------------------------------------------------------------
  Thirteen families. Each writes a JSON sidecar and touches the consuming
  rule so Caster hot-reloads automatically. All accept an optional spoken
  word that pre-fills the first InputBox. (Broadcasts use a persistent
  Gui instead — see below.)

  site add | add site [<textnv>]          — add a web-opener site (phrase +
                                             URL + title-part + default
                                             window position). URL and
                                             title auto-fill from the
                                             current Chrome tab. 4th box
                                             asks default position: blank
                                             Enter = focus-or-open; left/
                                             right/main/top/bot/twin =
                                             always snap a new window
                                             there when saying "<phrase>"
                                             alone. Alias detection if URL
                                             already saved under another
                                             phrase; dialog offers Alias
                                             (PgUp) / Replace (PgDn) /
                                             Cancel (Esc). Case-insensitive
                                             phrase match to existing entry
                                             defaults box to the EXISTING
                                             phrase (press Enter to update
                                             instead of creating a lowercase
                                             duplicate).

  spot add | add spot [<textnv>]          — add a Spotify/YouTube slot
                                             phrase (phrase + SpotLinks.ini
                                             slot key). If the slot isn't
                                             in SpotLinks.ini yet, offers
                                             to populate with the current
                                             Chrome URL.

  swallow add | add swallow [<textnv>]    — add a hardcoded spit slot
                                             (phrase + slot name). After
                                             saving, offers to view/edit
                                             the encrypted slot content in
                                             VS Code (decrypt -> edit ->
                                             re-encrypt on OK). Enter
                                             defaults to "No — leave as-is".

  send add | add send [<textnv>]          — add a Claude prompt (phrase +
                                             file stem). Creates the .txt
                                             if new, offers to add a
                                             Stream Deck button on GECK2
                                             CLAUDE (Deck B, key-logic:
                                             tap=SendClaudePrompt,
                                             hold=EditClaudePrompt),
                                             opens the .txt in VS Code at
                                             the end so you can type the
                                             prompt body.

  project add | add project [<textnv>]    — add a VS Code project (phrase +
                                             folder or .code-workspace
                                             path). Path auto-fills from
                                             the most-recently-opened
                                             VS Code workspace (sqlite
                                             query against state.vscdb).

  note add | add note [<textnv>]          — add a Notepad++ file (phrase +
                                             file path). Path auto-fills
                                             from the foreground Notepad++
                                             window title.

  number add | add number [<textnv>]      — add a Google Voice contact
                                             (display name + E.164 phone
                                             like +15551234567). Scoped to
                                             the Google Voice tab. Display
                                             name auto-fills from the
                                             "Messages with X" thread
                                             heading (parentheticals
                                             stripped, first names only).
                                             Phone auto-fills from the
                                             URL's itemId= thread partner
                                             or via a UIA Text scrape on
                                             contacts.google.com.

  group add | add group [<textnv>]        — add a Google Voice group
                                             thread. Must be open on a GV
                                             group-thread tab — thread ID
                                             auto-pulls from itemId=g.<id>
                                             in the URL. Display name
                                             auto-fills from the "Messages
                                             with A, B, ..." heading as
                                             space-joined first names
                                             ("Izzy Natalia" for a two-
                                             person thread). One InputBox
                                             (display name only); thread
                                             ID is not user-editable.

  broadcast add | add broadcast [<textnv>] — add a Google Voice broadcast
                                             (a send-to-each-member-
                                             individually group). First an
                                             InputBox for display name,
                                             then a persistent Gui: filter
                                             box at top, ListView of all
                                             valid member names (contacts
                                             + group threads) on the left,
                                             "Added so far" list on the
                                             right. Type to filter, Enter
                                             adds the top match, double-
                                             click adds a specific one,
                                             Del removes from the added
                                             list, PgDn (or Ctrl+S) saves,
                                             Esc cancels. Opening with a
                                             name that already exists pre-
                                             populates the added list so
                                             the same flow edits existing
                                             broadcasts.

  members add | add members <broadcast>    — edit an existing broadcast.
                                             Skips the display-name
                                             InputBox and opens the Gui
                                             directly for <broadcast>
                                             (a Choice populated from
                                             gv_broadcasts.json keys, so
                                             only real broadcast names
                                             match — no accidental fires
                                             on contact names).

  link add | add link [<textnv>]          — add a saved-link category
                                             (phrase + PascalCase
                                             filename stem; stem defaults
                                             to PascalCase of the phrase
                                             so Enter usually suffices).
                                             The .txt under SavedLinks\
                                             isn't pre-created —
                                             SaveCurrentLink writes it
                                             on the first "save link
                                             <phrase>". Stem is validated
                                             against [A-Za-z0-9_-].

  music add | add music [<textnv>]        — add a music URL (phrase +
                                             direct URL). URL auto-fills
                                             from the current Chrome tab.
                                             Used by "spot <phrase>" and
                                             "lockout <phrase>". Shorter
                                             2-InputBox flow (phrase +
                                             URL only — no title, no
                                             default_pos, since music URLs
                                             always open directly via
                                             SpotOpenInChrome).

  genre add | add genre [<textnv>]        — add a Spotify genre (phrase +
                                             GenreLinks.ini section name).
                                             Used by "spot <genre>",
                                             "spot swallow <genre>", and
                                             "lockout <genre>". 2-InputBox
                                             flow: box 2 defaults to
                                             PascalCase of the phrase
                                             ("indie folk" -> "IndieFolk")
                                             so Enter usually suffices.
                                             Section is stubbed Count=0 in
                                             GenreLinks.ini if new. If
                                             Chrome is foreground with a
                                             valid URL, offers to swallow
                                             it into the new section in
                                             the same flow (same as
                                             "spot swallow <genre>" later).

  doc add | add doc [<textnv>]            — add a Google Doc (phrase + URL
                                             + title-part + default window
                                             position). URL and title auto-
                                             fill from the current Chrome
                                             tab (Google Docs/Sheets/Slides
                                             suffix stripped). 4th box asks
                                             for a default position: blank
                                             Enter = focus-or-open (normal
                                             tab); left/right/main/top/bot/
                                             twin = always snap a new
                                             window there when saying "doc
                                             <phrase>" alone. Explicit
                                             "doc <phrase> <pos>" always
                                             overrides. Alias detection same
                                             as site add. Case-insensitive
                                             phrase match to existing entry
                                             defaults box to the EXISTING
                                             phrase.

  stack add | add stack [<textnv>]        — add a hardcoded stack name
                                             (phrase + on-disk filename
                                             stem under TEXTDOCS\programming\
                                             stacks\<key>.txt). Key defaults
                                             to snake_case of the phrase so
                                             Enter usually suffices. Also
                                             auto-triggered when "switch
                                             stack <unknown>" is spoken —
                                             the unknown name is passed as
                                             the pre-filled phrase.

  JSON sidecars live under AutoHotkey/INIDATA/VoiceChoices/. Each add
  dispatch shells to AutoHotkey/Scripts/VoiceConfigManager/add_voice_choice.py
  which writes the JSON and updates the rule file's # _reload_marker line.

Stack
-----
  stack that                 — grab entire focused field, push to TOP of active stack (deletes field contents)
  stack that bottom          — same as "stack that" but push to BOTTOM of active stack (will pop last)
  stack that <key>           — cut focused field and push to top of the NAMED stack without switching active.
                                <key> is a hardcoded name (macros, research, programming, etc. — see
                                hardcoded_stack_slots.json). Unknown name ("stack that testing") tooltips
                                "No stack named 'testing'. Say 'add stack testing' to create it." — no cut.
  pop it                     — paste #1 (top) stack item into focused field
  pop it <n>                 — pop the Nth item from the top (e.g. "pop it three")
  stack push <n>             — move the Nth item to the BOTTOM of the stack
  stack <key>                — select-all + copy focused field into the named stack (COPY not cut, no switch)
  show stack                 — 10s tooltip of active stack with numbers (#1 = next to pop); Esc to dismiss
  open stack [key] [pos]     — open stack file in Notepad++ (items numbered, #1 at bottom = next to pop)
    pos: top / bot (default) / full
    e.g. "open stack", "open stack top", "open stack research", "open stack research top"
  save stack <key>           — copy current stack to named file (does not switch). Accepts any new name.
  switch stack <key>         — change active stack to a hardcoded name (e.g. "switch stack research").
                                Unknown name ("switch stack testing") auto-opens the Add Stack UI pre-
                                filled with the spoken word, so just hit Enter twice to hardcode it.
  stack add | add stack [<textnv>]
                             — add a new hardcoded stack name — see Voice-Driven Config Adds.
  dump stack                 — delete entire current stack
  list stack                 — show all stack names (current marked with *)
  Stack files: C:\Users\jamie\Desktop\Important\TEXTDOCS\programming\stacks\
  Hardcoded names: C:\Users\jamie\Desktop\Important\AutoHotkey\INIDATA\VoiceChoices\hardcoded_stack_slots.json
