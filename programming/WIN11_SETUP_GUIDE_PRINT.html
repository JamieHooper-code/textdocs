1.4 Touchscreen

Two monitors — only one is a touchscreen. Windows 11 sometimes assigns touch input to the wrong display.

Step 1: Check if Windows detects it

  • Open Device Manager (Win + X → Device Manager)
  • Look under Human Interface Devices for "HID-compliant touch screen"
  • If it's there with no warning icon, touch is likely working — test before going further

Step 2: If touch isn't working

  • Right-click the touchscreen entry → Update driver → Search automatically
  • If no entry exists: Action → Scan for hardware changes
  • If yellow warning icon: right-click → Uninstall device → restart

Step 3: If driver install fails

  • Go to the monitor manufacturer's website and download the driver manually
  • Or: Settings → Windows Update → Advanced options → Optional updates

Step 4: Calibrate if needed

  • Search "Calibrate the screen for pen or touch input" in Start

Step 5: If touch registers on the wrong monitor

  • Search "Tablet PC Settings" → Display tab → Setup → tap the correct screen
  • Then Calibrate → Touch to fine-tune accuracy

[Claude can help] If Device Manager shows the device but touch still isn't responding, share what the device entry says.

Dragon 15 will NOT work long-term on Windows 11. Dragon 15 uses the deprecated JournalPlaybackHook API. As of 24H2, dictation in browser address bars and many apps is already broken. Dragon 16 was rebuilt for Windows 11.

Buy Dragon Professional v16 (standalone desktop) — NOT "Dragon Professional Anywhere" (that's enterprise cloud).

Installer location: E:\Downloads\Important\Nuance Dragon Professional VLA 16.10.200.044 — already on E: drive.
  • Run the Dragon installer from E:\Downloads\Important\Nuance Dragon Professional VLA 16.10.200.044
  • Restore profile: copy %APPDATA%\Nuance\NaturallySpeaking15\ backup → place in %APPDATA%\Nuance\NaturallySpeaking16\
  • Run Dragon's microphone check
  • Test basic dictation before moving on
  • Do NOT install Natlink yet — come back to that in Phase 2.1
If you temporarily must use Dragon 15 on Windows 11:
Avoid Windows 11 feature updates. Known broken in 23H2+: dictation in Chrome address bar, PDFs, file rename dialogs. Workaround: use Dragon's "Dictation Box" and paste.

2.1 Python + Natlink + Caster

This section is the most technically involved part of the whole setup. Read it fully before starting. The order matters.

Why Python must be 32-bit: Natlink bridges Dragon (32-bit plugin API) to Python. Even on a 64-bit machine, Natlink and Caster must use 32-bit Python. A 64-bit Python can coexist for other things — just keep them separate.

Required: Python 3.10.x 32-bit

Step 1: Install Python 3.10 32-bit

  • python.org → Downloads → Python 3.10.x → "Windows installer (32-bit)"
  • Check "Add Python to PATH" during install
  • Verify: py -3.10-32 --version → Python 3.10.x

[Claude can help] Verify architecture: py -3.10-32 -c "import struct; print(struct.calcsize('P')*8)" → should print 32

Step 2: Install Natlink v5.5.7

  • Download from github.com/dictation-toolbox/natlink/releases (latest .exe installer)
  • Run installer — it will detect Dragon and Python 3.10 32-bit
  • Open Natlink config GUI (Start Menu → "Configure NatLink via GUI")
  • Click (re)Register NatLink
  • Restart Dragon — splash screen should mention Natlink loading
If Natlink doesn't register: usually wrong Python architecture (64-bit found instead of 32-bit), or Dragon wasn't fully closed. Close Dragon, re-run installer, re-register via GUI.

Step 3: Clone Caster repos from GitHub (SSH required)

  • Clone user-caster (rules + settings) to C:\Users\Jamie\AppData\Local\caster
  • Clone caster-main (Caster source) to C:\Users\Jamie\Documents\Caster

Step 4: Install Caster Python dependencies

  • py -3.10-32 -m pip install -r C:\Users\Jamie\Documents\Caster\requirements.txt
  • If wxpython fails: py -3.10-32 -m pip install wxpython (no pinned version)

[Claude can help] Debug any pip install failures.

Step 5: Update settings.toml

  • Open C:\Users\Jamie\AppData\Local\caster\settings\settings.toml
  • Replace all C:\Users\Nate → C:\Users\Jamie (USER_DIR, BASE_PATH, ENGINE_PATH, REBOOT_PATH, AHK_PATH, DLL_PATH, etc.)
  • Update ENGINE_PATH to Dragon 16: C:\Program Files (x86)\Nuance\NaturallySpeaking16\Program\natspeak.exe

[Claude can help] Do the full search-and-replace on settings.toml — just say "fix settings.toml paths".

Step 6: Verify rules are enabled

  • Open C:\Users\Jamie\AppData\Local\caster\settings\rules.toml
  • Confirm all rule names are listed in _enabled_ordered

[Claude can help] Compare rules.toml list against actual .py files in rules/ folder.

Step 7: Test Caster

  • Start Dragon first, then Caster (or use startup program)
  • Say the test phrase from the first rule in any file (e.g., "Example Commands are working")
  • If a rule fails, Dragon/Natlink Messages window will show the Python error

[Claude can help] Debug any rule load errors — most are import path issues or settings.toml path problems.
